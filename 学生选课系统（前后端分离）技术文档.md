## 3. 环境搭建
### 3.1 基础环境安装
#### 3.1.1 JDK 17
- 下载地址：[Oracle JDK 17](https://www.oracle.com/java/technologies/downloads/#java17) 或 [OpenJDK 17](https://jdk.java.net/17/)
- 配置：安装后配置 `JAVA_HOME` 环境变量，将 `%JAVA_HOME%/bin`（Windows）/ `$JAVA_HOME/bin`（Linux/macOS）加入系统PATH。
- 验证：终端执行 `java -version`，输出JDK 17版本信息即为成功。

#### 3.1.2 Node.js 18.x
- 下载地址：[Node.js 官网](https://nodejs.org/zh-cn/download/)（选择18.x LTS版本）
- 验证：终端执行 `node -v`（输出v18.x.x）、`npm -v`（输出9.x.x）即为成功。

#### 3.1.3 Docker & Docker Compose(初学者可以省略这个)
- 安装参考：
  - Windows/macOS：安装 [Docker Desktop](https://www.docker.com/products/docker-desktop/)（内置Compose）
  - Linux：参考 [Docker 官方文档](https://docs.docker.com/engine/install/)
- 验证：终端执行 `docker -v`、`docker compose version`，输出对应版本信息即为成功。

### 3.2 数据库环境（Docker方式）
项目推荐通过Docker快速启动MySQL，无需手动安装配置：
1. 进入项目根目录；
2. 执行以下命令启动MySQL容器：
   ```bash
   docker compose up -d
   ```
3. 容器启动后自动创建如下配置：
   - 数据库名：`scs`
   - 用户名：`root`
   - 密码：`root`
   - 端口：默认3306（可在docker-compose.yml中修改）
4. 验证：使用MySQL客户端连接 `localhost:3306`，输入账号密码可访问即为成功。

## 4. 项目启动
### 4.1 后端启动
#### 方式1：使用Gradle Wrapper（推荐，无需手动装Gradle）
```bash
# 进入后端目录
cd backend
# Windows系统
./gradlew.bat bootRun
# Linux/macOS系统
./gradlew bootRun
```
#### 方式2：手动安装Gradle后启动
```bash
cd backend
gradle bootRun
```
- 启动成功验证：
  - 健康检查：访问 `http://localhost:8080/actuator/health`，返回 `{"status":"UP"}`；
  - 接口文档：访问 `http://localhost:8080/swagger-ui/index.html`，可看到完整接口文档；
  - 初始管理员：系统首次启动自动创建，账号 `admin@example.com`，密码 `Admin123`。

### 4.2 前端启动
```bash
# 进入前端目录
cd frontend
# 安装依赖
npm install
# 启动开发服务
npm run dev
```
- 前端配置说明：
  - 前端默认对接后端地址 `http://localhost:8080`；
  - 如需自定义后端地址，在 `frontend/.env.local` 文件中配置：
    ```env
    VITE_API_BASE_URL=http://自定义后端IP:端口
    ```
- 启动成功：终端输出前端访问地址（如 `http://127.0.0.1:5173/`），访问后可看到系统页面。

## 5. 核心功能接口说明
### 5.1 课程CSV导入（管理员专属）
- 接口地址：`POST /api/admin/courses/import`
- 请求方式：POST
- 请求类型：multipart/form-data
- 字段名：`file`（上传CSV文件）
- CSV文件格式要求：
  | 列名         | 说明                            | 必填 |
  | ------------ | ------------------------------- | ---- |
  | name         | 课程名称                        | 是   |
  | teacherEmail | 教师邮箱（需提前存在）          | 是   |
  | credits      | 课程学分                        | 是   |
  | capacity     | 课程容量（可选人数）            | 是   |
  | dayOfWeek    | 上课星期（1-7，1=周一）         | 是   |
  | startPeriod  | 开始节次                        | 是   |
  | endPeriod    | 结束节次                        | 是   |
  | location     | 上课地点                        | 是   |
  | schedule     | 展示用上课时间（如“周一3-4节”） | 是   |
  | description  | 课程描述                        | 否   |
- CSV示例：
  ```csv
  name,teacherEmail,credits,capacity,dayOfWeek,startPeriod,endPeriod,location,schedule,description
  高等数学,t1@example.com,4,120,1,1,2,A101,周一1-2节,微积分基础
  大学物理,t2@example.com,3,100,3,3,4,B202,周三3-4节,力学+电磁学
  ```

## 6. 安全配置说明
1. JWT密钥配置：
   - 路径：`backend/src/main/resources/application.properties`
   - 配置项：`app.jwt.secret`
   - 要求：**必须替换为自定义的超长随机字符串**（建议32位以上），避免默认密钥导致安全风险。
2. Token机制：
   - Access Token：JWT格式，用于接口认证，时效性短；
   - Refresh Token：存储于数据库，支持轮换机制，用于刷新Access Token，提升安全性。

## 8. 常见问题
1. 后端启动失败：
   - 检查JDK版本是否为17；
   - 检查MySQL容器是否启动，端口是否被占用；
   - 检查 `application.properties` 中数据库配置是否正确。
2. 前端启动后无法调用后端接口：
   - 检查后端是否启动，地址是否正确；
   - 检查浏览器跨域报错，后端已配置CORS，若仍有问题需确认 `VITE_API_BASE_URL` 配置；
3. CSV导入失败：
   - 检查CSV格式是否符合要求（列名、字段类型、必填项）；
   - 检查教师邮箱是否已在系统中存在；
   - 检查文件编码为UTF-8，避免中文乱码。
